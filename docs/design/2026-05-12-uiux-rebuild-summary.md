# 迷惑行为 UI/UX 重构总结

创建日期：2026-05-12

## 产品定位

迷惑行为编辑台是给公众号编辑团队使用的 mobile-first 协作后台。核心任务是把飞书多维表格同步来的投稿整理成不同期数，编辑对投稿和吐槽语进行点赞、采用、排序和预览。

## 信息架构

- 顶部栏：左侧显示产品标题“迷惑行为”，右侧是同步入口和个人头像。
- 头像菜单：显示当前用户，预留账户管理入口，支持退出登录。
- 主切换：保留两个 tab，分别是“投稿库”和“期数”。
- 投稿库：展示所有投稿，按飞书唯一序号倒序排列，不允许手动排序。
- 期数：展示当前期数内的投稿，允许上移/下移排序，后续可增强为长按拖拽。
- 期数管理：在期数页顶部通过当前期数名和小箭头打开，可切换期数、新建期数、重命名期数。
- 底部固定栏：保留筛选入口；将导出替换为“预览”。

## 期数规则

- 默认新建期数名为 `2026.5.12 期` 这类日期格式。
- 期数名可完全自定义。
- 一个投稿只能属于一个期数。
- 投稿库内展示投稿当前归属：`已加入 xxx` 或 `未分配`。

## 投稿卡片

- 卡片采用主流社交软件的信息流样式。
- 图片是投稿的第一视觉焦点。
- 图片区域固定为三列九宫格，超过九张继续向下扩展。
- 学校、授权状态、投稿时间位于同一信息层。
- 归属期数单独一行展示。
- 飞书序号不在卡片中展示，只作为投稿库默认排序依据。
- 投稿人原始吐槽语不作为投稿正文展示，只进入吐槽区。
- 投稿点赞使用爱心，不再使用星标。
- 投稿互动行从左到右为：爱心与数量、全部点赞人名字横向滚动、最右侧采用勾选。
- 投稿勾选表示最终采用，采用后会进入预览/导出内容。

## 吐槽区

- 每条投稿下方都有吐槽区。
- 吐槽展示为精简的 `名字：吐槽语`，不显示头像。
- 吐槽语勾选表示最终采用该吐槽语，预览/导出时使用它。
- 每条投稿最终最多采用一条吐槽语，再次点击已采用吐槽可取消采用。
- 飞书投稿人原始吐槽会入库为一条 `投稿人吐槽语`，参与点赞和采用。
- 投稿人吐槽语不可被普通编辑删除。
- 编辑只能编辑或删除自己的吐槽语。
- 如果自己的吐槽语已被采用，删除时提示需先取消采用。
- 删除自己的吐槽语前始终二次确认。

## 权限

第一版所有编辑权限相同：可点赞、采用、取消采用、排序期数内投稿、新建/切换/重命名期数、预览。

## 筛选与预览

- 筛选入口固定在底部，对投稿库和当前期数都适用。
- 筛选包含学校、投稿时间、飞书序号范围、授权状态、归属期数、采用状态。
- 筛选面板支持将当前草稿筛选结果批量加入某一期，后端使用单次事务处理。
- 预览按钮打开 Figma 风格的当前期数文章预览页。
- 原 HTML 导出 API 继续保留为次级能力。

## 当前实现状态

- 前端主界面以 Figma 导出页面为视觉真源，顶部栏、主 tab、期数栏、底部栏、头像菜单、期数面板、筛选面板、投稿卡外壳、同步弹窗和预览页均按 Figma 代码迁移。
- 投稿卡已接真实数据和 server actions：学校编辑、期数加入/重新分配/移除、投稿点赞、投稿采用、No. 排序、吐槽点赞/采用/编辑/删除。
- 图片支持九宫格展示、全屏预览、启用/停用、手动补图；导出和预览只使用启用图片。
- 飞书同步已从同步确认弹窗触发真实同步逻辑，并记录 SyncJob；同步时只刷新飞书来源图片，保留手动补图和图片启用状态。
- 当前未完成：账户管理、长按拖拽排序、预览页内复制/正式导出操作、同步任务队列和重试。
