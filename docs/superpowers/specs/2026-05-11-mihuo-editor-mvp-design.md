# 迷惑行为编辑台 MVP 设计说明

创建日期：2026-05-11

## 产品目标

建立一个 mobile-first 的编辑后台，用于把飞书多维表格里的投稿同步到本地数据库，供约 30 名公众号编辑筛选、投票、确认、排序、写吐槽语，并导出微信公众号文章 HTML。

## 第一版范围

- 账号密码登录，不做权限区分。
- 所有编辑拥有同等操作权限。
- 用户头像优先用用户名首字或首字母生成。
- 仅编辑后台，不做粉丝公开页。
- 先本地运行，暂不依赖服务器、域名或 HTTPS。
- 飞书 API 第一阶段先预留同步结构，本地用种子数据跑通业务流。
- 基础 HTML 导出先跑通内容结构，微信公众号样式规范后续迭代。

## 技术选型

- React + Next.js App Router
- TypeScript
- Prisma ORM
- SQLite 用于本地开发，后续部署可切 PostgreSQL
- shadcn/ui + lucide-react 做成熟组件和图标
- Tailwind CSS 做布局和设计 token
- bcryptjs 做密码哈希
- jose 做本地 JWT session cookie
- sharp 预留图片压缩能力

## 视觉方向

参考图：`docs/design/mobile-editor-concept.png`

- Mobile first，手机屏宽优先。
- 白色/浅中性背景，避免营销页和大装饰。
- 扁平、紧凑、轻边框。
- 绿色表示最终确认，琥珀色表示星标喜欢。
- 使用 icon button、底部操作栏、紧凑 tabs。
- 投稿信息以列表/轻卡呈现，不做复杂嵌套卡片。

## 核心数据模型

- `User`：编辑用户。
- `Submission`：投稿主记录，保存飞书记录 ID、学校、投稿时间、原始吐槽、同意状态、状态等。
- `SubmissionImage`：投稿图片，保存原始地址、本地地址、压缩信息。
- `Issue`：公众号推送篇目。
- `IssueItem`：篇目内投稿及排序、是否确认采用。
- `Comment`：编辑给投稿写的吐槽语。
- `Reaction`：星标喜欢，支持投稿和吐槽语。
- `Selection`：确认采用，支持投稿和吐槽语；吐槽语同一投稿只能最终确认一条。
- `SyncJob`：飞书同步任务记录。

## MVP 页面

1. `/login`：账号密码登录。
2. `/app`：移动端编辑台主界面。
3. `/app/issues/[id]`：某篇推送的编辑页，第一版可先合并在 `/app` 的 tab 内。
4. `/api/export/[issueId]`：导出基础 HTML。

## MVP 工作流

1. 用户登录。
2. 进入编辑台，看到投稿库、本期、导出三个 tab。
3. 投稿库可查看本地种子投稿。
4. 编辑可以对投稿点星标，或勾选确认纳入本期。
5. 编辑可以给投稿新增吐槽语。
6. 编辑可以对吐槽语点星标。
7. 编辑可以为每条投稿确认唯一最终吐槽语。
8. 本期内可调整投稿顺序。
9. 导出页生成基础 HTML。

## 后续飞书同步设计

- 使用飞书记录 ID 去重。
- 手动同步优先，自动同步后置。
- 每次同步只拉新增或变更记录。
- 图片下载后本地缓存并压缩。
- 每条同步任务记录成功数、失败数、错误信息和耗时。

## 已确认决策

- 第一版不做权限区分。
- 第一版先账号密码登录。
- 吐槽语最终采用只能选一条。
- 投稿确认后进入导出。
- 全员可确认投稿、确认吐槽、排序、导出。
- UI 使用成熟组件和 icon，避免大量手写 UI。
